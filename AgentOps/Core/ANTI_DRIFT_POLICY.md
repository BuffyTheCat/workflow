# Anti-Drift Policy

## Hard Rules

- No scope expansion.
- No opportunistic refactor.
- No touching unrelated files.
- No changing tests to fit broken behavior.
- No invented facts.
- No pretending ticket, MCP, browser, or media context was checked when it was not.
- No hiding uncertainty behind fluent prose.

## Behavioral Rules

- Keep edits minimal.
- Preserve user constraints across long runs and context compaction.
- Call out degraded mode explicitly.
- Stop and escalate when evidence is too weak for safe progress.
